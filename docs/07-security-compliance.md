# 07 — Безопасность и соответствие требованиям (GDPR)

## 1. Аутентификация и авторизация
- **JWT**: короткоживущий access-токен (~15 мин) + refresh-токен (ротация, в БД
  хранится только хэш; при использовании старый инвалидируется — защита от повторного
  использования). Logout/«выйти на всех устройствах» = ревокация refresh-токенов.
- **OAuth 2.0 / OIDC**: Google и Microsoft; связка через `oauth_account`.
- **Пароли**: bcrypt (cost ≥ 12); политика сложности; защита от перебора (rate-limit
  + временная блокировка).
- **RBAC**: роли `tutor | student | parent | admin`; guard'ы на эндпоинтах и проверка
  владения ресурсом (репетитор видит только своих учеников; ученик — только свои уроки).
- **2FA** (TOTP) — в бэклоге, желательно до публичного запуска.

## 2. Защита данных
- **В транзите**: TLS 1.2+ везде (HTTPS/WSS), HSTS.
- **В покое**: шифрование диска БД и объектного хранилища; чувствительные поля
  (токены Telegram, платёжные идентификаторы) — отдельная защита/секрет-менеджер.
- **Секреты**: только в переменных окружения / secret manager; не в репозитории
  (`.env` в `.gitignore`, есть `.env.example`).
- **Платёжные данные карт не хранятся** — токенизация на стороне Stripe/PayPal
  (снижает область PCI DSS до SAQ-A).

## 3. Безопасность приложения
- Глобальная валидация входных данных (class-validator, whitelist) — защита от
  инъекций/некорректных полей; Prisma (параметризованные запросы) — против SQLi.
- Rate-limiting (Redis) на auth и публичных эндпоинтах; CORS allow-list.
- Защита заголовками (helmet), CSRF-меры для cookie-сессий, ограничение размера/типа
  загружаемых файлов, антивирус-скан загрузок (бэклог).
- Подпись вебхуков Stripe/PayPal проверяется обязательно; идемпотентность обработки.
- Аудит-лог (`audit_log`) ключевых действий (вход, платежи, изменение ролей, доступ к
  записям уроков).

## 4. GDPR / приватность (рынки ЕС/UK)
- **Правовые основания и согласия** (`consent`): ToS, Privacy, отдельно маркетинг;
  версионирование текстов.
- **Права субъекта данных**:
  - доступ/экспорт — выгрузка персональных данных пользователя (JSON/PDF);
  - исправление — редактирование профиля;
  - удаление («right to be forgotten») — каскадная анонимизация/удаление с учётом
    юридических сроков хранения финансовых документов.
- **Минимизация данных**: собираем только необходимое; записи уроков — с явного
  согласия участников, индикатор записи виден всем.
- **Хранение в ЕС** (data residency): регион БД/хранилища — EU; реестр
  суб-обработчиков (LiveKit, Stripe, PayPal, email-провайдер) с DPA.
- **Несовершеннолетние**: роль родителя и родительское согласие для учеников младше
  установленного возраста.
- **Сроки хранения**: политика retention (логи, записи уроков, финансы) + автоудаление.

## 5. Надёжность и эксплуатация
- Резервное копирование БД (PITR) и восстановление по расписанию; проверка бэкапов.
- Мониторинг (OpenTelemetry, Sentry), алёрты, SLO на доступность урока.
- План реагирования на инциденты и уведомление об утечках в течение 72 ч (GDPR).
- Изоляция окружений (dev/staging/prod), принцип наименьших привилегий для доступа.
